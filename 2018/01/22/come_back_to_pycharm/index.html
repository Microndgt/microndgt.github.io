弃坑VSCode 拥抱PyCharm

2018-01-22 07:47:05
  • 编程

因为新鲜使用了一段VSCode，可是觉得还是有很多不如人意的地方，还是觉得PyCharm好用，失去了才知道珍惜么 = =! 哈哈，总结一下，VSCode有下面的一些问题：

  1. Git diff功能支持的不好，像PyCharm上可以查看某个Commit甚至多个Commit的diff，但是VSCode上装什么插件好像都没法这样看，只能一个文件一个文件的看，很麻烦，或许是我没找到怎么弄，但是毕竟我也读了一些git diff插件的文档，但是还没有找到，所以说明也不好用。
  2. Git插件已经add或者提交了，目录树上还是绿的黄的一片，不更新。。这个很蛋疼，强迫症的我总以为有东西还没提交。
  3. 在大量debug模式下，有时候就没法跳转函数定义啦，自动补全也失效了，这个也很蛋疼，只能重新打开这个项目。
  4. 没法Debug Celery的tasks，只能用rdb一行一行的看，效率很低。

基于上面原因，VSCode实在没法继续用了，不过转投PyCharm有个硬伤，快捷键。。哎，只好苦练了。下面就是Mac下的PyCharm 快捷键总结。

另外用Goland写Golang应该还是不错的，现在我就觉得，就应该使用专业工具，通用工具虽然吊，但是bug很多。老牌的专业工具还是稳定好用。
